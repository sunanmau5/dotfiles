# Writing Style

- Never use en dashes (–) or em dashes (—). Use a plain hyphen (-) if a dash is
  needed
- Keep prose concise and direct
- Single-sentence bullet points do not end with a period
- When unsure about implementation details, asks user questions using the
  `request_user_input` tool

# Always-On Coding Guidelines

Apply the Karpathy Guidelines in every coding chat:

- Think before coding: state assumptions, surface ambiguity, and ask when unclear
- Prefer simplicity: write the minimum code that solves the request
- Keep changes surgical: touch only what the task requires and match existing style
- Clean up only artifacts introduced by your own changes
- Define concrete success criteria and verify the result with relevant checks
